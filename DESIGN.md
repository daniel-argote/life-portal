# Life Portal Design Principles

## 1. Radical Customizability
The user should be able to mold the interface to fit their mental model, not the other way around.
- **Rearrangeable UI:** Navigation, dashboard widgets, and lists should be draggable/reorderable.
- **Personalized Identity:** Users can rename any module or header to match their own vocabulary.
- **Granular Control:** Provide toggles for UI elements (like headers) that some may find redundant but others find essential.
- **Aesthetic Agency:** Deep support for themes and visual styles.

## 2. Frictionless Capture
Recording data (logs, food, health, money) should take as few clicks/taps as possible.

## 3. Data Ownership
The system is built on Supabase, ensuring users have a clear path to their own data.
